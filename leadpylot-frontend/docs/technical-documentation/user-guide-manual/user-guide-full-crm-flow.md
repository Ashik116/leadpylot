**📘 Overview — index — LeadPylot CRM — User Guide**

- **Introduction**
    
    LeadPylot is a  CRM built for teams that manage leads, offers, and deals from first contact through to closed contract.  Agents work their assigned leads, build and send offers, and track each deal through a structured pipeline. Admins oversee the full operation — configuring the system, managing users, importing leads, and monitoring every stage of the pipeline. The system also includes integrated email, a Kanban task board, PDF contract generation, meetings and calendar, and AI assists in capturing lead information through conversations using the built-in LeadBot.
    
    ---
    
- **Role Overview**
    
    
    | Role | Responsibilities | Has Access To |
    | --- | --- | --- |
    | **Admin** | Manages all users, configures the system, imports and assigns leads, oversees the full offer and deal pipeline, and tracks agent performance across all projects | Everything — all leads, all agents, all configuration settings, all reports |
    | **Provider** | The provider has access to their reclaimed leads and associated email communications. | Their own reclaimed leads and email account |
    | **Agent** | Works their assigned leads day-to-day — updates lead status, creates offers, manages deals through the pipeline, and communicates with leads via email | Only leads assigned to them; their own offers, openings, mails, tasks, and calendar |

    **Notes**
    
    - The browser talks primarily to **Next.js** (SSR, App Router). The app calls backend services via **Axios** (interceptor stack).
    - **NextAuth v5** handles sessions on the frontend tier; services validate **JWT** and optional **gateway** headers.
    - **Socket.IO** (notification service) pushes real-time updates to the client.

  - **4. Admin Setup Checklist**
    Complete these steps in order before agents begin working. Each step depends on the previous one being in place.
    1. **Configure Banks** — Go to **Configuration → Banks**. Add each bank that agents will use when creating offers. Set the bank name, type, and amount limits.
    2. **Add Mail Server(s)** — Go to **Configuration → Mail Servers**. Add at least one IMAP/SMTP mail server. Agents need a mail server to send and receive emails from within the CRM.
    3. **Create Lead Sources** — Go to **Configuration → Sources**. Add the lead sources you use (e.g. Google Ads, Cold Call). Set a price per lead for cost tracking.
    4. **Configure Lead Stages** — Go to **Configuration → Stages**. Create the stages that describe where a lead is in the sales process (e.g. New, Warm, Hot, Termin). Set the order and a color for each.
    5. **Set Up PDF Templates** — Go to **Configuration → PDF Templates**. Upload your fillable PDF contract files and map the CRM fields (contact name, amount, bank, etc.) to the correct fields in each PDF.
    6. **Create User Accounts** — Go to **Configuration → Users**. Create an account for each agent and provider who will use the system. Assign each user a role and an office.
    7. **Set Role Permissions** — Go to **Configuration → Users → Permissions**. Review and adjust the feature access for each role. Confirm that agents can only see their own assigned leads unless you want broader visibility.
    8. **Create Projects** — Go to **Configuration → Project Settings**. Create at least one project. A project links together a group of agents, the banks they offer, a mail server, and PDF templates.
    9. **Assign Agents to Projects** — Open each project and add the agents who will work leads in that project. Set each agent’s alias name, username, and mail server assignment within the project.
    10. **Import Leads** — Go to **Configuration → Import** (or click **Import** in the top navigation). Upload a CSV or Excel file of leads. Select the import type and source. The system processes the file and adds the leads.
    11. **Assign Leads to Agents** — Go to **Leads → Leads Bank**. This shows all unassigned leads waiting for distribution. Select leads and use bulk actions to transfer them to agents.
        > **Note:** The CRM is ready for agents to use once Steps 1–9 are complete. Steps 10–11 can be repeated at any time as new leads come in.

- **🛡️ User Management** · `admin/02-user-management.md`
  **Section:** Admin Guide
  **Applies to:** Admin role only
  ***
  - **Quick Summary**
    - LeadPylot has three roles: **Admin**, **Provider**, and **Agent** — each with different levels of access.
    - Admins create and manage all users from **Configuration → Users**.
    - Projects group users for data scoping and team-level visibility.
    - The **Permissions** page controls what each role can see and do across the entire CRM.
    - Permission changes apply to **all users with that role immediately** — no per-user overrides.
    ***
  - **1. The Three Roles**
    ### Admin
    Full access to everything. Admins can see all leads, all agents, all data across the system. They manage users, configure system settings, create projects, import leads, access reporting, and control permissions. There is no feature in the CRM that is off-limits to an Admin.
    ### Provider
    Providers see their own leads. They can manage the work of agents assigned to them, but they cannot see data belonging to other providers. They cannot access system-wide configuration settings.
    ### Agent
    A frontline role. Agents work only with leads assigned to them. They can create offers, manage their own pipeline, and communicate with their leads. They cannot see other agents’ leads or access any configuration area.
    | Role | Scope | Can Configure System? |
    | ------------ | ----------------------- | --------------------- |
    | **Admin** | All data, all users | Yes |
    | **Provider** | Own leads’s data only | No |
    | **Agent** | Own assigned leads only | No |
    ***
  - **2. Create a User**
    Only Admins can create users.
    1. Go to **Configuration** → **Users** → click **Create User**.
    2. Fill in the form:
       - **Full Name** — user’s display name in the system.
       - **Email** — used as the login identifier and for system notifications.
       - **Login** — username for logging in (can match the email or be a separate username).
       - **Password** — set a temporary password. The user should change it on first login.
       - **Role** — select **Admin**, **Provider**, or **Agent**.
       - Project — assign the user to an project group (see [Section 4](about:blank#4-offices)).
       - **Alias Name** — short display name shown in lead cards, emails, and call logs.
       - **Mail Server** — the default outgoing mail account for this user.
       - **VOIP Username** — the SIP username for VoIP calling. Leave blank if the user does not make calls.
    3. Click **Save**.
       The user can now log in immediately using the email and password you set.
       > **Tip:** Inform new users can not change their password from **My Profile.**
    ***
  - **3. Edit or Deactivate a User**
    1. Go to **Configuration** → **Users** → **All Users**.
    2. Find the user in the list.
    3. Click **Edit** on the user’s row.
    4. Update any fields as needed.
    5. To disable the account without deleting it, delete button the user status to **Inactive**.
       An **Inactive** user cannot log in. Their historical data (leads, emails, offers) remains intact. Reactivate the account at any time by setting the status back to **Active**.
    ***
  - **4. Offices**
    Offices are groupings of users that control data visibility. Agents in the same office may share certain data views depending on how permissions are configured.
    ### Create an Office
    1. Go to **Configuration** → **Users** → **Offices**.
    2. Click **Add Office**.
    3. Enter an **Office Name**.
    4. Click **Save**.
    ### Assign a User to an Office
    Assign the office when creating a new user or by editing an existing user.
    A user can only belong to one office at a time. Changing a user’s office affects which team data they can see going forward — it does not retroactively change historical records.
    ***
  - **5. Permissions**
    The Permissions page controls what each role can do across the CRM. Changes here affect **every user with that role** instantly.
    ### Navigate to Permissions
    Go to **Configuration** → **Users** → **Permissions**.
    ### How the Permissions Matrix Works
    The page displays a grid:
    - **Rows** = individual feature permissions (e.g., “Create Offer”, “Delete Lead”, “Access Reporting”).
    - **Columns** = roles (Admin, Provider, Agent).
    - Each cell contains a checkbox. A checked box means that role has that permission.
      Toggle any checkbox to grant or revoke a permission for a role. There is no separate save step — changes apply immediately.
    ### Permission Categories
    | Category                 | What It Controls                                                              |
    | ------------------------ | ----------------------------------------------------------------------------- |
    | **Lead Visibility**      | Whether a role can see all leads, only their own leads, or their team’s leads |

- **🛡️ Offer Pipeline** · `admin/05-offer-pipeline.md`
  - **Quick Summary**
    - **Offer Creation** — Agents create offers from leads. Admins review and promote them into the pipeline by creating an Opening.
    - **Pipeline Stages** — An Opening moves through: Contract → Confirmation → Payment Vouchers → Netto 1 → Netto 2 → Deal Closed.
    - **Revert** — Available at every stage. Moves the opening back one step without deleting any data.
    - **Split & Commission** — Each opening can have split amounts across agents, individual commission percentages, admin commission, and a bonus amount.
    - **Bulk Stage Moves** — Select multiple offers or openings and use **Switch To** to move them to the next stage in one action.
    ***
  - **Section 1: Pipeline Overview**

    ```
    OFFER CREATED (by Agent)
        ↓
    SEND TO OPENING  ← begins the pipeline
        ↓
    CONTRACT  (signed contract document uploaded)
        ↓
    CONFIRMATION  (deal details confirmed)
        ↓
    PAYMENT VOUCHERS  (payment initiated)
        ↓
    NETTO 1  (first net amount verification)
        ↓
    NETTO 2  (second net verification)
        ↓
    [DEAL CLOSED]

    LOST  ← can be triggered from any stage
    ↑ REVERT ← available at every stage to go back one step
    ```

    An offer starts at the Offer stage and progresses through the pipeline as the deal moves closer to completion. At every stage, you can **Revert** the opening to the previous stage, or send it to **Lost** if the deal falls through.

  - **Section 4: Revert**
    **Revert** is available at every stage of the pipeline.
    - Revert moves the opening back to the **previous stage**.
    - Use Revert when a document is rejected, deal details need correction, or the client requests changes before proceeding.
    - Revert does **not** delete the opening. It simply steps the opening back one position in the workflow.
    - All previously uploaded documents and entered data are preserved after a Revert.
      **Example:** An opening currently at **Netto 1** → click **Revert** → the opening moves back to **Payment Vouchers**.
      To revert, select the opening, click **Switch To**, and choose **Revert**.

  - **Section 5: Financial Fields**
    ### Split Amount
    When a deal is worked by more than one agent, the admin can split the deal amount between them.
    - Open the opening **Edit** form.
    - Add a split entry for each agent involved.
    - Each split entry contains: agent name | their portion of the total amount | their commission percentage.
    - The sum of all split amounts should equal the total offer amount.
    ### Agent Commission
    - The percentage of the net deal amount earned by the agent who worked the lead.
    - Set in the opening **Edit** form (for example, 3% of the amount).
    - The commission value is calculated automatically once the percentage is entered.
    ### Admin Commission
    - A platform-level commission tracked separately from the agent commission.
    - Visible and editable in the opening **Edit** form.
    - Used to record the margin retained by the platform on the deal.
    ### Bonus Amount
    - A fixed bonus added on top of the agent’s calculated commission.
    - Configured by the Admin under **Configuration → Bonus Amount**.
    - The bonus can be set per bank or applied globally across all deals.
    - Once configured, the bonus is applied automatically to qualifying openings.
    ### Payment Terms
    - Defines the loan duration for the deal (for example, 12 months, 24 months, or 36 months).
    - Set on the offer at the time of creation.
    - Available payment term options are configured by the Admin under **Configuration → Payment Terms**.

    4. The result is a fully completed contract document ready for signature

    4. For each PDF field, use the dropdown to select the matching CRM data field:
       | PDF Field Example | Map To CRM Field |
       | ------------------------------ | ----------------- |
       | Full Name / Vollständiger Name | Contact Name |

       | Duration / Months | Payment Terms |
       | Payment Frequency | Payment Frequency |

       | Date | Current Date |
       | Signature | Signature Field |

    6. Click **Save Mapping**

  - **Sources & Stages** — control where leads come from and how they progress through your sales pipeline
  - **Payment Terms & Bonus Amounts** — configure offer durations and agent commission bonuses
  - **VOIP Servers & Bot Settings** — connect your FreePBX calling infrastructure and AI/Telegram bots
  - **Security & Table Settings** — enforce password policies, session limits, and default column layouts
  ***
  - **Section 1: Banks**
    **Configuration → Settings → Banks**
    Banks are the financial institutions and product types available when agents create offers for leads.
    **What each bank entry contains:**
    - **Name** — e.g. Gold Bank
    - **Type** — Fixed Deposit, Loan, Mortgage, Investment, etc.
    - **Minimum Amount** — the floor investment or loan amount
    - **Maximum Amount** — the ceiling investment or loan amount
      **To add a bank:**
    1. Click **Add Bank**.
    2. Enter the **Name**, **Type**, **Minimum Amount**, and **Maximum Amount**.
    3. Click **Save**.
       **To edit a bank:**
    4. Locate the bank row in the table.
    5. Click the pencil (edit) icon.
    6. Update the fields and click **Save**.
       > **Note:** Agents see only the banks assigned to their project when creating an offer. Assign banks to projects under **Projects → Project Settings**.
       > **Example:** Gold Bank – Fixed Deposit, minimum €10,000, maximum €500,000.
    ***
  - **Section 2: Sources**
    **Configuration → Settings → Sources**
    Sources track where each lead originates. They are used in import batches and in cost-per-lead reporting.
    **What each source entry contains:**
    - **Name** — e.g. Google Ads, Partner Network, Cold Call, Referral
    - **Price per Lead** — the cost attributed to each lead from this source, used in import cost reports
      **To add a source:**
    1. Click **Add Source**.
    2. Enter the **Name** and **Price per Lead**.
    3. Click **Save**.
       **To edit a source:**
    4. Click the pencil icon on the source row.
    5. Update the fields and click **Save**.
       > **Usage:** On the **Import** page, agents select which source an import batch belongs to. The source price is applied automatically to all leads in that batch and appears in cost analysis reports.
    ***
  - **Section 3: Stages**
    **Configuration → Settings → Stages**
    Stages define the sales progression a lead moves through, from first contact to closing or rejection.
    **What each stage entry contains:**
    - **Name** — e.g. New, Warm, Hot, Appointment, NE1, NE2, NE3, NE4
    - **Order Number** — determines the position of the stage in the pipeline sequence
    - **Color** — displayed as a colored indicator on lead rows in the leads table
      **To add a stage:**
    1. Click **Add Stage**.
    2. Enter the **Name**, **Order Number**, and select a **Color**.
    3. Click **Save**.
       **To reorder stages:**
    4. Drag any stage row up or down to change its position.
    5. The order number updates automatically.
       **To edit a stage:**
    6. Click the pencil icon on the stage row.
    7. Update the fields and click **Save**.
       > **Usage:** Agents update a lead’s current stage from the lead detail page. The stage color appears in every table that displays leads, giving agents a quick visual status at a glance.

  - **Section 4: Payment Terms**

    **Examples:** 12 months, 24 months, 36 months, 60 months.

    1. Click **Add Payment Term**.
    2. Enter the duration label (e.g. `24 months`).
    3. Click **Save**.
       **To edit a payment term:**
    4. Click the pencil icon on the term row.
    5. Update the label and click **Save**.

    5. Click the pencil icon on the bonus row.

- **👤 Openings & Pipeline** · `agent/04-openings-pipeline.md`
  **Quick Summary:**
  - **What an opening is** — when an Admin promotes an offer into the deal pipeline, it becomes an **Opening** (an active deal moving through formal stages)
  - **Pipeline stages** — **Contract** → **Confirmation** → **Payment Vouchers** → **Netto 1** → **Netto 2** → deal closed; **Lost** can apply from any stage
  - **Revert** — step back one stage without deleting the opening or its documents (available at every stage)
  - **Document uploads** — upload required files per stage (contract, SWIFT, N1 Mail, N2 Mail) from the **Openings** table columns
  - **Split amount** — in **Edit**, use **Add Split** to share a deal between agents; each portion and commission is tracked separately

  ***
  - **Section 1: What Is an Opening?**
    When an offer is promoted (by Admin) into the deal pipeline, it becomes an **Opening**.
    An **Opening** represents a deal that is actively progressing through formal stages — from contract signing through net amount verification. Each stage requires specific documents and confirmations.
    An Opening is linked to: its original offer, the lead, the agent, and the bank.
    ***
  - **Section 2: The Pipeline Stages**

    ```
    OFFER
      ↓ (Promoted to Opening by Admin)
    CONTRACT  ← upload signed contract document here
      ↓
    CONFIRMATION  ← confirm deal details are correct
      ↓
    PAYMENT VOUCHERS  ← payment processing initiated
      ↓
    NETTO 1  ← first net amount verification; upload N1 Mail
      ↓
    NETTO 2  ← second net verification; upload N2 Mail
      ↓
    [DEAL CLOSED]

    LOST  ← can be triggered from any stage if deal falls through
    ↑ REVERT  ← available at every stage to step back one stage
    ```

    The deal starts as an offer. After Admin promotes it, the opening sits in **Contract** until you upload the signed contract. You then move forward through **Confirmation** (checking details), **Payment Vouchers** (payment processing and SWIFT), and **Netto 1** / **Netto 2** (two net-amount checks with the corresponding mail documents). When **Netto 2** is complete, the deal is treated as closed. If the deal fails at any point, it can be marked **Lost**. **Revert** is always available to go back one stage — for example, to fix a wrong document or update details — without starting the deal over.

    ***

    3. Update the required fields → click **Save**
    ***
  - **Section 8: Split Amount**
    When a deal is shared between you and another agent:
    1. In the edit form → find the **Split** section
    2. Click **Add Split** → select the other agent → enter their portion of the amount
    3. Each split entry shows: **Agent** | **Amount** | **Commission %**
    4. The total of all split amounts should equal the full deal amount
    5. Click **Save** — each agent’s commission is tracked separately

  - **Mail inbox** — folders for Inbox, Sent, Drafts, Starred, All Mail, Pending, Opening, and Assigned Agent; list shows sender, subject, time, and lead badge
  - **Read and reply** — open any email for full view, thread history, and attachments; **Reply** and **Forward** open the compose panel with context

    | **All Mail**       | Inbox + Sent combined               |

    | **Assigned Agent** | Emails assigned to you as the agent |

    - Sender name and email

    - Received time
    - Lead badge: **No Lead** (not linked) or lead name (linked)

    1. Click any email in the inbox. The full view opens.

    ***
  - **Section 3: Reply and Forward**
    - **Reply**: click **Reply**. The compose panel opens with the recipient and thread context filled in.
    - **Forward**: click **Forward**. The compose panel opens ready to forward to a new recipient.

    2. Fill in the fields:
       - **To**: recipient email address
       - **Subject**: email subject line
       - **Body**: write in the rich text editor (bold, italic, bullet lists, links)

- **👤 Tasks & Todos** · `agent/07-tasks-todos.md`
  **Quick Summary:**
  - **Kanban board overview** — columns (**To Do**, **In Progress**, **In Review**, **Done**) show all your work items in one place
  - **Auto-task creation** — new leads, offers, openings, and assigned mail spawn linked task cards without manual setup
  - **Drag-and-drop stage updates** — move cards between columns to change status; other agents get notified when their cards move
  - **Predefined tasks** — Admin templates add checklist subtasks to new cards (for example, follow-up steps for new leads)
  - **Lead todos and tickets** — use the **Tasks** tab on a lead for simple todos and escalated **Tickets** for Admin follow-up
  ***
  - **Section 1: What Is the Kanban Board?**
    Navigate to **Kanban** in the top navigation.
    The Kanban board is a task management view for all your work items:
    - Each **column** represents a work stage: **To Do** | **In Progress** | **In Review** | **Done**
    - Each **card** represents a task linked to a lead, offer, opening, or email
    - Supports **Dark Mode**: click the dark mode toggle in the top-right to switch themes
    - Shows **all** your tasks in one place — you do not need to open each lead separately
    ***
  - **Section 2: Auto-Task Creation**
    Tasks are created automatically when key events happen in the CRM:
    | Trigger Event | Auto-Task Created |
    | ------------------------------------- | ----------------------- |
    | New lead created / assigned to you | Lead task in the Kanban |
    | Offer created | Linked offer task |
    | Opening created | Linked opening task |
    | Email received and assigned to a lead | Linked mail task |
    Each auto-task is linked to its source entity. Click the task card to open the linked lead, offer, or opening.
    ***
  - **Section 3: Kanban Card Details**
    Each card on the Kanban board shows:
    | Element | Description |
    | ------------------ | ------------------------------------------------------------------------------------------------ |
    | **Title** | Lead name, offer description, or task name |
    | **Badge** | **Lead** / **Offer** / **Opening** / **Mail** — shows what type of entity this card is linked to |
    | **Assigned Agent** | Avatar of the agent responsible |
    | **Due Date** | When this task should be completed |
    | **Priority** | Indicator (high, medium, low) |
    | **Subtask Count** | Number of predefined subtasks on this card |
    ***
  - **Section 4: Drag-and-Drop Stage Update**
    1. Find the task card you want to move.
    2. Click and hold the card.
    3. Drag it to the target column (for example, from **To Do** to **In Progress**).
    4. Release — the card’s stage updates immediately.
    5. If the card is assigned to another agent, they receive a notification.
    ***
  - **Section 5: Create a Task Manually**
    1. Click the **+** button in any Kanban column.
    2. Fill in the task form:
       - **Title**: task name
       - **Description**: additional details
       - **Due Date**: deadline
       - **Assigned Agent**: select who should do this
       - **Linked Lead** (optional): attach this task to a specific lead
       - **Priority**: **High** | **Medium** | **Low**
    3. Click **Save** — the card appears in that column.
    ***
  - **Section 6: Assign an Agent to a Task**
    - On any task card: click the **agent avatar** icon.
    - Select a different agent from the dropdown.
    - The selected agent receives an in-app notification (and Telegram notification if configured).
    - The task now appears on the assigned agent’s Kanban board.
    ***
  - **Section 7: Predefined Tasks (Admin-Configured)**
    Admin sets up task templates in **Configuration → Predefined Tasks**. These templates automatically attach a set of subtasks to new Kanban cards.
    **Example — “New Lead” template:**
    - [ ] Call the lead
    - [ ] Send introduction email
    - [ ] Schedule follow-up appointment
    - [ ] Create offer
          When a new lead is assigned to you, the Kanban card for that lead automatically includes these subtasks. Check each one off as you complete it.
    ***
  - **Section 8: Todos in Lead Detail**
    You can also manage tasks directly from a lead’s detail page without going to the Kanban board:
    1. Open a lead → click the **Tasks** tab.
    2. All todos linked to this lead appear here.
    3. Click **+** to add a new todo: enter description and due date → **Save**.
    4. Check the box next to a todo to mark it complete.
    5. Todos also appear on the lead row in the leads table (in the active todos count).
    ***
  - **Section 9: Tickets**
    A **Ticket** is an escalated task that requires Admin attention:
    1. From the lead detail page, raise a ticket using the **Ticket** button in the **Tasks** tab.
    2. Describe the issue or request.
    3. Admin receives the ticket in their task view.
    4. Admin assigns it to the right person and tracks resolution.
